# ReviewClassifier

**ReviewClassifier** — программный комплекс для воспроизводимой обработки, классификации и анализа пользовательских рецензий с применением методов машинного обучения и обработки естественного языка.

Проект воспроизводит техническую часть исследования **«Будущее в технологическом воображении массовой аудитории: текст-майнинг зрительских рецензий на научно-фантастические фильмы»**  
**Суслов И. В., Кныжова З. З., Олифиренко А. А.**

Программный комплекс обеспечивает сборку и проверку набора данных, сегментацию рецензий, отбор технологически релевантных высказываний, подготовку ручной разметки, обучение классификаторов, классификацию текстовых фрагментов, агрегацию результатов и построение тепловых карт.

## Назначение проекта

Программа предназначена для автоматизации полного цикла анализа пользовательских рецензий:

- подготовки корпуса текстовых данных;
- выделения смысловых фрагментов;
- отбора технологически релевантных высказываний;
- классификации фрагментов по заданным категориям;
- анализа динамики оценочных суждений;
- формирования таблиц, отчётов и визуализаций.

Проект используется в задачах текст-майнинга, анализа пользовательского мнения, цифровой гуманитаристики, исследования массового восприятия технологий, а также в учебных и научно-исследовательских проектах по машинному обучению и обработке естественного языка.

## Структура репозитория

Основные каталоги проекта:

```text
data/
├── raw/                         # исходные Excel-файлы
├── processed/                   # фрагменты после сегментации и отбора
├── labels/                      # ручная или слабая разметка
├── results/                     # отчёты, таблицы и предсказания
└── figures/                     # тепловые карты и визуализации

models/                          # обученные модели
src/                             # исходный код программы
docs/                            # методические материалы и инструкции
legacy_collection_scripts/       # скрипты сбора HTML и офлайн-парсинга

## Установка
Склонируйте репозиторий и перейдите в каталог проекта:
```bash
git clone git@github.com:PanOlifer/ReviewClassifier.git
cd ReviewClassifier
```
```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

На macOS при проблемах с установкой `torch` установите PyTorch по официальной инструкции для вашей версии Python и процессора, затем повторно выполните остальные команды.

## Основной pipeline

```bash
python run_pipeline.py
```

Основной pipeline выполняет:

1. проверку исходного набора данных;
2. сегментацию рецензий;
3. отбор технологически релевантных фрагментов;
4. обучение базового классификатора, если используется текущий режим `run_pipeline.py`;
5. классификацию фрагментов;
6. агрегацию по годам и произведениям;
7. построение тепловых карт;
8. лексический анализ.

Файлы основных результатов:

- `data/results/dataset_inspection.xlsx`;
- `data/processed/fragments_all.csv`;
- `data/processed/fragments_tech.csv`;
- `data/results/classification_fragment_level.csv`;
- `data/results/reviewer_tables.xlsx`;
- `data/results/yearly_polarity.csv`;
- `data/results/film_polarity.csv`;
- `data/figures/heatmap_yearly_polarity_2005_2025.png`;
- `data/figures/heatmap_film_polarity_top20.png`.


## Полный запуск с PyTorch-обучением

При выполнении основного пайплайна затем отдельного обучения PyTorch-классификатора, используйте:

```bash
python run_pipeline_with_pytorch.py
```

Основные таблицы и тепловые карты будут созданы обычным pipeline. PyTorch-модель будет обучена после этого и сохранена отдельно.

## Подготовка ручной разметки

```bash
python make_annotation_sample.py
```

После запуска появится файл:

```text
data/labels/annotation_sample_for_manual_coding.xlsx
```

В нём нужно заполнить столбец `label` значениями:

```text
optimism
skepticism
mixed
```

После разметки сохраните файл как:

```text
data/labels/labeled_fragments.xlsx
```

Если файл `data/labels/labeled_fragments.xlsx` существует, программа использует его вместо слабой словарной разметки.

## Обучение PyTorch-модели

PyTorch-модель запускается отдельно и не входит в основной `run_pipeline.py`.

Минимальный запуск:

```bash
python -m src.train_pytorch_classifier
```

Запуск с предварительной проверкой данных и подготовкой фрагментов:

```bash
python run_pytorch_training.py
```

После обучения появятся:

```text
models/pytorch_embeddingbag_classifier.pt
models/pytorch_vocab.json
data/results/pytorch_classification_report.txt
data/results/pytorch_training_metadata.json
```

## Применение PyTorch-модели ко всем фрагментам

```bash
python -m src.predict_fragments_pytorch
```

Скрипт создаёт отдельный файл:

```text
data/results/classification_fragment_level_pytorch.csv
```

Этот файл не используется основным pipeline и не перезаписывает `data/results/classification_fragment_level.csv`.

## Архитектура PyTorch-модели

Модель реализована как компактный классификатор коротких текстовых фрагментов:

- токенизация текста регулярным выражением;
- построение словаря по обучающей части;
- `EmbeddingBag` для усреднённого векторного представления фрагмента;
- линейный классификационный блок;
- вероятностный выход по классам `optimism`, `skepticism`, `mixed`;
- функция потерь `CrossEntropyLoss`;
- оптимизатор `Adam`.


Для повышения качества проверки классификационного компонента добавлен отдельный полярный PyTorch-классификатор:

```bash
python -m src.train_pytorch_polar_classifier
```

Он решает задачу `optimism` vs `skepticism` после исключения класса `mixed`.

Чтобы выполнить сразу все:

```bash
python run_pipeline.py
python -m src.train_pytorch_classifier
python -m src.predict_fragments_pytorch
python -m src.train_pytorch_polar_classifier
```

## Важно: обучение на 5000 размеченных фрагментов

Скрипты обучения читают файл `data/labels/labeled_fragments_5000.xlsx` с приоритетом над старым `data/labels/labeled_fragments.xlsx`. Это сделано, чтобы модель не обучалась случайно на старой разметке из 1000 строк.

Проверить и установить 5000-разметку как основную можно командой:

```bash
python install_5000_labels.py
```

После этого запускайте обучение:

```bash
python -m src.train_baseline
python -m src.train_pytorch_classifier
python -m src.train_pytorch_polar_classifier
```

В начале обучения должно быть видно:

```text
Используется файл разметки: .../data/labels/labeled_fragments_5000.xlsx
Всего валидных размеченных фрагментов: 5000
```

## Воспроизводимость

Для воспроизводимого запуска рекомендуется:

- использовать виртуальное окружение Python;
- не изменять структуру каталогов `data/`, `models/`, `src/`;
- сохранять исходные данные в `data/raw/`;
- хранить ручную разметку в `data/labels/`;
- запускать основной pipeline из корневого каталога проекта;
- фиксировать версии зависимостей в `requirements.txt`.
